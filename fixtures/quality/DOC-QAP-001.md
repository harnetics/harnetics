# 天行一号运载火箭质量管理计划

## 1. 概述

### 1.1 目的

本文档建立天行一号（TH-1）运载火箭研制过程中的质量管理体系，规定质量保证活动的要求和流程，确保产品质量满足系统级需求（DOC-SYS-001）和客户要求。

### 1.2 适用范围

本计划适用于天行一号运载火箭从方案设计到首飞的全研制过程，覆盖所有分系统和组件。

### 1.3 引用文件

- DOC-SYS-001 天行一号运载火箭系统级需求文档 v3.1
- GJB 9001C 质量管理体系要求
- GJB 1391A 故障模式、影响及危害性分析
- GJB 900A 系统安全性通用大纲

### 1.4 职责分工

| 角色           | 职责                                     |
| -------------- | ---------------------------------------- |
| 质量与可靠性部 | 质量体系管理、质量审计、问题归零监督     |
| 各分系统设计部 | 设计质量保证、评审参与、问题整改         |
| 试验与验证部   | 试验质量控制、数据审核、设备校准         |
| 技术负责人     | 技术决策、放行审批                       |

## 2. 质量管理体系

### 2.1 可靠性设计准则

全系统可靠性指标：单次发射可靠性 ≥ 0.95（DOC-SYS-001 REQ-SYS-002）

**设计准则**：
1. 关键单机冗余设计（DOC-SYS-001 REQ-SYS-020）
2. 降额设计：电子元器件降额 ≥ 50%
3. 余量设计：结构安全系数 ≥ 1.25（极限载荷）/ ≥ 1.5（屈服载荷）
4. 最坏情况分析：关键参数按 3σ 边界分析

### 2.2 设计评审节点

| 阶段       | 评审名称               | 评审内容                     | 输出物           |
| ---------- | ---------------------- | ---------------------------- | ---------------- |
| 方案设计   | 方案评审（SDR）        | 系统方案、指标分配           | 评审报告         |
| 初步设计   | 初步设计评审（PDR）    | 初步设计方案、接口定义       | 评审报告         |
| 详细设计   | 关键设计评审（CDR）    | 详细设计、FMEA、试验方案     | 评审报告         |
| 生产       | 试验就绪评审（TRR）    | 产品合格证、试验大纲         | 放行单           |
| 试验       | 试验结论评审           | 试验数据、偏差处理           | 评审报告         |
| 首飞       | 飞行就绪评审（FRR）    | 全系统验证状态               | 飞行许可         |

### 2.3 文档管理要求

1. 所有技术文档必须有唯一编号（使用 DOC-XXX-NNN 格式）
2. 文档版本管理遵循语义化版本号（vX.Y）
3. 跨部门引用的文档必须注明引用版本号
4. 文档更新后必须通知所有引用方进行一致性检查
5. ICD（DOC-ICD-001）为受控文档，变更需经技术负责人批准

## 3. 试验质量控制

### 3.1 试验前要求

1. 试验大纲经评审批准（评审记录存档）
2. 所有测量设备在校准有效期内（参见 §5）
3. 试验件完成入所检验
4. 试验前技术交底会议
5. 安全预案就绪

### 3.2 试验数据要求

1. 数据采集系统双机备份
2. 原始数据不可修改（HDF5 格式锁定）
3. 数据处理过程可追溯
4. 数据异常需在 24 小时内填写《试验数据异常报告》

### 3.3 试验后要求

1. 试验报告在试验完成后 5 个工作日内提交
2. 试验报告需包含与判定标准的逐项对比
3. 不合格项需触发 §4 归零流程

## 4. 质量问题管理

### 4.1 问题分级

| 级别     | 定义                                           | 处理时限     |
| -------- | ---------------------------------------------- | ------------ |
| I 类     | 影响任务成败或人员安全                         | 7 个工作日   |
| II 类    | 影响产品性能但不影响任务成败                   | 14 个工作日  |
| III 类   | 一般性质量问题                                 | 30 个工作日  |

### 4.2 归零流程

所有 I 类和 II 类质量问题必须执行"双五条归零"流程：

**技术归零五条**：
1. 定位准确
2. 机理清楚
3. 故障复现
4. 措施有效
5. 举一反三

**管理归零五条**：
1. 过程清楚
2. 责任明确
3. 措施落实
4. 严肃处理
5. 完善规章

### 4.3 归零报告

归零报告需经以下人员签字：
- 问题发生方技术负责人
- 质量代表
- 项目总师（I 类问题）

## 5. 计量与校准

### 5.1 校准要求

| 设备类型     | 校准周期 | 校准基准                |
| ------------ | -------- | ----------------------- |
| 力/推力传感器 | 6 个月   | 国家二级标准力传感器     |
| 压力传感器   | 6 个月   | 活塞式压力计（二等）     |
| 温度传感器   | 12 个月  | 标准铂电阻温度计         |
| 流量传感器   | 6 个月   | 标准容积法               |
| 振动传感器   | 12 个月  | 标准振动台               |

### 5.2 校准管理

- 建立校准台账，记录校准日期和到期日
- 超期未校准的设备禁止用于试验
- 校准证书存档，试验报告附校准证书编号

---

## 6. 供应商质量管理

### 6.1 合格供应商名录

- 关键供应商需通过现场审核
- 每年复审一次

### 6.2 进货检验

- 关键原材料：100% 批检
- 一般物料：抽样检验（AQL 1.0%）

## 7. 配置管理

### 7.1 配置项

以下文档纳入配置管理：
- DOC-SYS-001 系统级需求文档
- DOC-ICD-001 全局接口控制文档

### 7.2 变更控制

ICD 和系统需求变更流程：
1. 变更方提交《工程变更申请单》（ECA）
2. 影响评估：识别受影响的下游文档和部门
3. 技术评审委员会审批
4. 变更实施
5. 变更验证
6. 更新受影响文档
7. 通知所有受影响方
