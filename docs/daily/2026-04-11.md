## 记录
- [决策] 在 `llm.client` 内支持裸 Ollama 模型名归一化，避免把 LiteLLM provider 细节泄漏给调用方。
- [决策] 对非 Ollama 的 `HARNETICS_LLM_BASE_URL` / `HARNETICS_EMBEDDING_BASE_URL`，允许 `.env` 使用 bare 模型名，并在运行时统一归一化到 `openai/<model>`。
- [决策] 通过 `HARNETICS_LITELLM_DEBUG=1` 控制 `litellm._turn_on_debug()`，只在边界层启用一次，避免把常规运行日志变成噪声瀑布。
- [决策] `POST /api/draft/generate` 改为显式使用 `app.state.settings` 注入 `HarneticsLLM`，不再依赖默认构造路径。
- [决策] 章节定位同时修 `graph/indexer.py` 和 `engine/impact_analyzer.py`：入库生成 section-aware 边，分析时 edge-first，旧图 fallback 到锚点匹配。
- [决策] `EmbeddingStore` 对 bare cloud embedding 模型（如 `text-embedding-3-small`）做显式归一化与远程路由判断，CLI `ingest --with-embeddings` 同步透传 `.env` 中的 embedding API key/base URL。
- [决策] `GET /api/documents/search` 从“向量不可用即 503”改为“AI 向量优先 + 关键词降级”，并显式返回 `analysis_mode` 与 `relevance_score`。
- [决策] 影响报告跳转草稿台时，不再只传 `doc_ids`，而是传 `report_id/source_report_id/trigger_doc_id/impacted_doc_id/new_version`，由草稿台自动生成主题并预选来源文档。
- [经验] 仅检查 `/api/tags` 是否 200 不足以证明 Ollama 可用；必须校验目标模型名是否出现在返回列表中。
- [经验] LiteLLM 对 `jina-embeddings-v5-text-small` 这类 bare 远程模型名不会自动推断 provider；只要 base URL 指向 OpenAI-compatible 网关，就该在边界层显式补成 `openai/<model>`。
- [经验] 如果 UI 或数据库里的 `generated_by` 仍写死本地模型名，观察层就会制造“请求还在走本地”的假象；必须持久化真实 effective model 才能可靠 debug。
- [经验] 只靠 upstream internal section id 做字符串匹配无法定位章节，因为下游文档不会出现 `DOC-XXX-sec-N` 这种内部 ID。
- [经验] `load_dotenv()` 的隐式搜索在测试和 CLI 场景下会捡到错误的 `.env`；应显式加载 `cwd/.env` 才有稳定语义。
- [经验] Chroma 自定义 embedding function 只实现 `__call__` 不够，查询路径还会要求 `embed_query/embed_documents/is_legacy/name/get_config/build_from_config`。
- [进展] 新增 `tests/test_llm_client.py`，并扩展 `tests/test_e2e_mvp_scenario.py` 锁定路由配置透传与 section-aware impact 定位；全量 pytest 通过。
- [进展] 验收 `004-ai-vector-impact-draft` 时补齐 `source_report_id` 合同字段、草稿台相关度标签、状态错误详情与搜索 fallback；按 31 + 27 的两组回归覆盖全量 58 个测试用例。
- [进展] 新增 `tests/test_env_routing.py`，用本地 fake OpenAI-compatible provider 端到端锁定 `.env` + bare 模型名 + completion/embedding 双路径，避免真实云端凭证影响回归稳定性。
