# Security

本项目的安全核心不是复杂加密术，而是边界清楚。

当前边界：
- 目标部署形态是本地运行，不把工程文档上传到外部 SaaS
- 仓库目前不保存密钥、不调用外部 API，也没有多租户边界
- `fixtures/` 中的数据应视为受控样本，未来真实文档导入必须走显式权限路径

实现红线：
- 未经明确设计，不得把文档内容自动发送到公网模型或第三方分析服务
- 日志不得默认落全文档正文
- 未来若引入模型权重、数据库或对象存储，必须补充访问控制与备份策略
